### Istnieje wiele elementów z rolą `"form"` bez unikalnych etykiet

<div id="locLevel"></div>

Jeśli na stronie istnieje wiecej niż jeden element z rolą `"form"`, każdy musi mieć unikalną etykietę opisującą jego przeznaczenie

</Column>

### Dlaczego to jest ważne

Element HTML `<form>` z etykietą lub tytułem, lub jakikolwiek element z rolą `"form"` jest typem regionu WAI-ARIA. Jeśli istnieje wiele obszarów `"form"`, muszą one mieć unikalne etykiety, aby osoby korzystające z technologii wspomagajacych mogły szybko zidentyfikować i&nbsp;nawigować do każdego formularza.


<div id="locSnippet"></div>

### Co zrobić

* Dla każdego elementu z rolą `"form" ustaw wartość atrybutu `aria-labelledby` na wartość `id` dowolnego widocznego tekstu, który odpowiednio i jednoznacznie opisuje zawartość tego obszaru;
* LUB użyj atrybutu „aria-label”, aby dodać niewidoczną, unikalną, niepustą etykietę do każdego elementu z rolą `"form"`.


Na przykład:

<CodeSnippet type="multi" light={true}>&lt;h2 id="na1"&gt; Nowe konto&lt;/h2&gt;
&lt;form  aria-labelledby="na1"&gt;
  &lt;fieldset&gt;
    &lt;legend&gt;Dane osobowe:&lt;/legend&gt;
    &lt;label for="imie"&gt;Imię:&lt;/label&gt;
    &lt;input type="text" id="imie" name="imie" value="Adam"&gt;
    &lt;label for="nazwisko"&gt;Nazwisko:&lt;/label&gt;&lt;br&gt;
    &lt;input type="text" id="nazwisko" name="nazwisko" value="Niebieski"&gt;&lt;br&gt;&lt;br&gt;
    &lt;input type="submit" value="Wyślij"&gt;
  &lt;/fieldset&gt;
&lt;/form&gt;
    
    
&lt;div role="form" aria-label="Adres wysyłki="&gt;
	...
&lt;/div&gt;</CodeSnippet>

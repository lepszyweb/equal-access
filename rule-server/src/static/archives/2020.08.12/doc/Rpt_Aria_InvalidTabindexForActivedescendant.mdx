### Element {0} używający właściwości `aria-activedescendant` ustawionej na {1} nie posiada wartości atrybutu `tabindex` ustawionej na 0 lub -1  

<div id="locLevel"></div>

Element używający właściwości `aria-activedescendant` musi mieć wartość atrybutu `tabindex` ustawioną na 0 lub -1, aby był dostępny z klawiatury.

</Column>
</Row>
<Row>
<Column colLg={11} colMd={5} colSm={4} className="toolMain">

### Dlaczego to jest ważne

Elementy używają właściwości `aria-activedescendant` do zarządzania fokusem ich aktywnych potomków. 
Ustawienie wartości tabindex na 0 lub -1 zapewnia, że element zostanie uwzględniony w sekwencji tabulacji i pozwala jego potomkom otrzymać fokus dla dostępu z klawiatury. Należy unikać używania wartości większych niż 0, ponieważ może to uszkodzić logiczną kolejność tabulacji.

<div id="locSnippet"></div>

### Co zrobić

* Ustaw w elemencie  wartość atrybutu `tabindex` równą 0 lub -1, używając właściwości `aria-activedescendant` aby włączyć element do sekwencji tabulacji. 

Poniższy przykład pokazuje element paska narzędzi używający właściwości `aria-activedescendant` z atrybutem tabindex ustawionym na zero: 

<CodeSnippet type="multi" light={true}>&lt;div role="toolbar" aria-label="Pasek narzędziowy" tabindex="0" aria-activedescendant="button2" id="tb3" onkeypress="keyEvent();"&gt;
    &lt;div id="button1" role="button1"&gt;Narzędzie 1&lt;/div&gt;
    &lt;div id="button2" role="button2"&gt;Narzędzie 2&lt;/div&gt;
&lt;/div&gt;  </CodeSnippet>

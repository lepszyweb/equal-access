### Element `<object>` nie ma tekstu alternatywnego

<div id="locLevel"></div>

Elementy `<object>` muszą posiadać alternatywę tekstową, która opisuje treść odwzorowaną przez obiekt

</Column>
</Row>
<Row>
<Column colLg={11} colMd={5} colSm={4} className="toolMain">

### Dlaczego to jest ważne

Opisy alternatywne są ważne dla użytkowników, których przeglądarki nie obsługują elementu `<object>` lub typu medium osadzonego za pomocą `<object>`.


<div id="locSnippet"></div>

### Co zrobić

* Jeśli element `<object>` jest apletem Java, dodaj alternatywny opis tekstowy;
* LUB, jeśli element `<object>` jest filmem MPEG, dodaj obraz z tekstem alternatywnym.

Na przykład:

<CodeSnippet type="multi" light={true}> &lt;object classid="java:Press.class" width="500" height="500" title="Aplet Java: Jak temperatura wpływa na ciśnienie"&gt;
&lt;applet code="java:Press.class" width="500" height="500"&gt; Wraz ze wzrostem temperatury wzrasta prędkość cząsteczek w balonie...
&lt;/applet&gt;
&lt;/object&gt;</CodeSnippet>

</Column>
<Column colLg={5} colMd={3} colSm={4} className="toolLeft">

### O tym wymaganiu

* [1.1.1 Treść nietekstowa](https://wcag.lepszyweb.pl/#non-text-content)
* [Technika WCAG 2.1: H53](https://www.w3.org/WAI/WCAG21/Techniques/html/H53)

### Na kogo to wpływa

* Osoby niewidome korzystające z czytników ekranu;
* Osoby korzystające z przeglądarek tekstowych (np. Lynx) lub interfejsów audio.

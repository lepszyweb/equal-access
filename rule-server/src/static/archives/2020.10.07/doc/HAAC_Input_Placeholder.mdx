### Dlaczego to jest ważne

Atrybut `<placeholder>` użyty jako jedyna widoczna etykieta zmniejsza dostępność dla szerokiego grona użytkowników z następujących powodów:
  * nie jest trwały; tekst zastępczy znika, gdy użytkownik zaczyna pisać w polu formularza;
  * można go pomylić ze wstępnie wypełnioną wartością;
  * kolor tekstu zwykle nie spełnia minimalnego współczynnika kontrastu;
  * brak etykiety zmniejsza obszar aktywny (trafienia) przy ustawianiu fokusu na pole wejściowe.
  
  

<div id="locSnippet"></div>

### Co zrobić

  * Użyj atrybutu `placeholder` jako krótkiej wskazówki (słowa lub krótkiej frazy), aby pomóc użytkownikowi we wprowadzaniu danych
  * ORAZ użyj prawidłowej metody etykietowania, która jest zarówno widoczna, jak i określona programowo;
  * LUB, jeśli nie można dodać widocznej etykiety, sprawdź, czy etykieta pola wejściowego jest zrozumiała w kontekście;
  

Na przykład:

<CodeSnippet type="single" light={true}>&lt;label&gt;Adres e-mail: &lt;input type="email" name="adres1" placeholder="jan@przyklad.net"&gt;&lt;/label&gt;</CodeSnippet>

<CodeSnippet type="single" light={true}>&lt;label for="adres2"&gt;Adres&lt;/label&gt;</CodeSnippet>

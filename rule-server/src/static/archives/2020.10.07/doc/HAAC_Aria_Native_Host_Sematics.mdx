### Dlaczego to jest ważne

Niektóre elementy lub atrybuty HTML mają rdzenną semantykę, która domyślnie mapuje się do zakladanej semantyki WAI-ARIA. Dla przewidywalnej interakcji i zachowania kontrolek interfejsu użytkownika ta domyślna semantyka może być nadpisywana tylko na określone sposoby, zdefiniowane w specyfikacji [W3C ARIA in HTML Document conformance requirements for use of ARIA attributes in HTML](https://www.w3.org/TR/html-aria/#docconformance).

<div id="locSnippet"></div>

### Co zrobić

* Jeśli rdzenny element HTML pasuje do zamierzonego zachowania, użyj tego elementu;
* LUB popraw wartość atrybutu role lub atrybutu WAI ARIA zgodnie ze wskazówkami sekcji specyfikacji W3C ARIA w HTML [Document conformance requirements for use of "ARIA" attributes in HTML](https://www.w3.org/TR/html-aria/#docconformance). Ta speyfikacja dostarcza szczegółów na temat poprawnych znaczników ARIA na natywnych elementach HTML.

Poniższy przykładowy kod poprawnie używa elementu `<aside>` do oznaczenia obszaru treści uzupełniającej na stronie.


<CodeSnippet type="single" light={true}>&lt;aside&gt;O wydawcy Max Perkins: Wydawca Geniusa (1978),…… &lt;/aside&gt; </CodeSnippet>
        
The following code correctly overrides the `&lt;aside&gt;` element’s implicit semantics (`role=“complementary”`) by using the ARIA role of `“note”` to designate the notated content on the page.

Poniższy kod poprawnie zastępuje niejawną semantykę elementu `&lt;aside&gt;` (`role="complementary"`) poprzez użycie roli ARIA `"note"`, aby oznaczyć  na stronie notatkę o treści.
    
<CodeSnippet type="single" light={true}>&lt;aside role="note"&gt;O wydawcy Max Perkins: Wydawca Geniusa (1978),…… &lt;/aside&gt; </CodeSnippet>

### Atrybut(y) `{0}`, do których odwołuje się element `<{1}>`, nie jest prawidłowym stanem lub właściwością WAI-ARIA

Atrybuty WAI-ARIA muszą być poprawne dla roli elementu

### Dlaczego to jest ważne

Gdy elementom przypisywana jest `role` WAI-ARIA, WAI-ARIA 1.1 określa, które atrybuty są prawidłowe dla tej roli. Jeśli używane są nieprawidłowe atrybuty WAI-ARIA, technologie wspomagające mogą nie być w stanie dokładnie interpretować i obsługiwać element dla użytkowników.



<div id="locSnippet"></div>

### Co zrobić

* Usuń stan lub atrybuty właściwości WAI-ARIA, które nie są prawidłowe dla roli elementu;
* LUB zmień wartość `role` na rolę, która obsługuje zamierzone zachowanie elementu, używając najnowszej specyfikacji WAI-ARIA jako przewodnika.

W poniższym przykładzie element odwołuje się tylko do prawidłowych atrybutów WAI-ARIA dla elementu z rolą `"tree"`: globalnej właściwości `aria-labelledby` i stanu `aria-expanded`.


Na przykład:

<CodeSnippet type="multi" light={true}>&lt;ul id="tree1" class="tree root-level" role="tree" aria-labelledby="label_1"&gt;
    
&lt;li id="owoce" class="tree-parent" role="treeitem" tabindex="0" aria-expanded="true"&gt;&lt;span&gt;Owoce&lt;/span&gt;</CodeSnippet>

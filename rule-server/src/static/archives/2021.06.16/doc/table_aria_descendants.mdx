### Jawne określenie roli WAI-ARIA 'role' nie jest poprawne dla elementu `<{0}>` w elemencie z rolą WAI-ARIA '{1}' zgodnie ze specyfikacją ARIA w HTML

<div id="locLevel"></div>

Elementy struktury tabeli nie mogą określać jawnie roli w kontenerach tabeli

</Column>

### Dlaczego to jest ważne?

Specyfikacja WAI-ARIA zapewnia zbiór ról dostępności jako główny wskaźnik typu kontrolki. Jest to używane do obsługi interfejsów API dostępności platformy. Użycie prawidłowej roli WAI-ARIA w elemencie zdefiniowanym w specyfikacji WAI-ARIA pozwala technologiom wspomagającym technologie wspomagające mogą wykorzystać semantykę roli do prezentacji i wsparcia spójnej interakcji z obiektem.

<div id="locSnippet"></div>

### Co zrobić

* Usuń wszelkie definicje ról z elementów struktury tabeli (`tr`, `th`, `td`) znajdujących się w dowolnym kontenerze z rolą `table`, `grid` lub `treegrid`. Poniższy przykład pokazuje element z poprawną `role="grid"` z odpowiednimi strukturami tabel:



<CodeSnippet type="multi" light={true}> 

&lt;div id="caption" class="caption"&gt;Podpis tabeli&lt;/div&gt;
&lt;table role="grid" aria-labelledby="caption" class="gridCls"&gt;
  &lt;thead&gt;
    &lt;tr&gt;
      &lt;th&gt;Nagłówek 1&lt;/th&gt;&lt;th&gt;Nagłówek 2&lt;/th&gt;
    &lt;/tr&gt;
  &lt;/thead&gt;
  &lt;tbody&gt;
    &lt;tr&gt;
      &lt;td&gt;Komórka z danymi 1&lt;/td&gt;&lt;td&gt;Komórka z danymi 2&lt;/td&gt;
    &lt;/tr&gt;
  &lt;/tbody&gt;
&lt;/table&gt;

</CodeSnippet>

</Column>

### Dlaczego to jest ważne

Tabele danych są używane do wyświetlania danych relacyjnych. Powiązanie komórek danych z odpowiednimi nagłówkami pozwala technologiom wspomagajacym na informowanie o nagłówkach dla każdej komórki danych, dzięki czemu użytkownik może zrozumieć złożone relacje danych tabelarycznych.

<div id="locSnippet"></div>

### Co zrobić

 * Jeśli nie jest to tabela danych (brak danych relacyjnych), użyj CSS zamiast elementu `<table>` do ułożenia treści (wysoce zalecane);
 * LUB określ tabelę jako tabelę ukłądu za pomocą `role="presentation"` lub `role="none"`;
 * LUB, jeśli jest to prosta tabela danych, użyj elementu `<th>` do identyfikacji nagłówków w pierwszym wierszu;
 * LUB, jeśli jest to złożona tabela danych, dodaj atrybut `scope` do elementu `<th>`, aby zidentyfikować nagłówki i określić zakres nagłówka dla każdej komórki danych;
 * LUB, jeśli jest to złożona tabela z wieloma wierszami lub kolumnami jako nagłówkami, użyj atrybutu `id` z elementem `<th>` i atrybutu `headers` z elementem `<td>`, aby określić wiele nagłówków dla każdej komórki danych (np. `<td headers="a1 a2">`).

### Dlaczego to jest ważne


Jeśli atrybut WAI-ARIA `role` jest określony dla elementu, muszą rownież istnieć wszystkie elementy potomne WAI-ARIA wymagane dla tej roli. Rodzime role WAI-ARIA elementów potomnych spełniają wymagania - rola nie musi być jawnie ustawiona. Bez wymaganych elementów podrzędnych technologie wspomagające mogą nie być w stanie dokładnie przedstawić elementu lub współdziałać z nim.


<div id="locSnippet"></div>

### Co zrobić

 * Dodaj odpowiednie wymagane elementy potomne do tego elementu.

Na przykład następujący element używa roli `"radiogroup"`. Rola `"radiogroup"` wymaga elementów potomnych z ustawieniem `role="radio"` w sposób reprezentujący zaznaczony lub niezaznaczony stan przycisku opcji.


<CodeSnippet type="multi" light={true}>&lt;div role="radiogroup"&gt;
    &lt;div role="radio" aria-checked="false"&gt;&lt;/div&gt;
    &lt;div role="radio" aria-checked="true"&gt;&lt;/div&gt;
&lt;/div&gt;</CodeSnippet>

Przykład z niejawną rolą:

<CodeSnippet type="multi" light={true}>&lt;table role= "treegrid"&gt;
     &lt;tr&gt;
         &lt;td&gt;
            element z role="treegrid" wymaga elementu potomnego z role="row".
            TR ma niejawną WAI-ARIA role="row", więc warunek jest spełniony
         &lt;td&gt;
     &lt;tr&gt;
&lt;/table&gt;</CodeSnippet>

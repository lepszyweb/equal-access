### Wartość `"{0}"` określona dla właściwości WAI-ARIA `{1}` dla elementu `<{2}>` jest nieprawidłowa

### Dlaczego to jest ważne

Kiedy elementom przypisany jest atrybut `role`, istnieją wymagane atrybuty dla tej roli. Atrybuty te reprezentują stany i właściwości elementu. Atrybuty WAI-ARIA akceptują określony zestaw tokenów jako wartości wraz z wartościami niejawnymi określanymi jako 'domyślne', gdy wartości nie są wskazane. Prawidłowa wartość atrybutu musi być określona, ponieważ dyktuje ona zachowanie kontrolki i jest używana przez programy użytkownika (przeglądarki) do przetwarzania zachowania kontrolki zgodnie z zaleceniami.

<div id="locSnippet"></div>

### Co zrobić

* Podaj poprawną wartość atrybutu zdefiniowanego dla elementu, używając najnowszej specyfikacji WAI-ARIA jako przewodnika.

Na przykład, następujący element akapitu definiuje właściwość `aria-live` z poprawną wartością `"polite"`.


<CodeSnippet type="multi" light={true}> 
&lt;p aria-live="polite"&gt;Treść obszaru z dymnamicznie zmienianą treścią (na żywo).&lt;/p&gt;</CodeSnippet>

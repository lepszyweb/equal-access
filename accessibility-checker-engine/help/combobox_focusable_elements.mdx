### Fokus klawiatury nie jest poprawnie ustawiony wewnątrz combobox lub powiązanym wyskakującym okienku

<div id="locLevel"></div>

Fokus klawiatury wewnątrz combobox można ustawić tylko na polu wprowadzania tekstu, z wyjątkiem sytuacji, gdy używane jest okienko dialogowe

</Column>
</Row>
<Row>
<Column colLg={11} colMd={5} colSm={4} className="toolMain">

### Dlaczego to jest ważne

Pole kombinowane (combobox) to interaktywny komponentem UI, który łączy dane wejściowe i inny element, taki jak pole listy lub siatka, który może dynamicznie wyskakiwać, aby pomóc użytkownikowi ustawić wartość wprowadzanych danych. Gdy pole kombi jest aktywowane, fokus powinien być ustawiony na elemencie tekstowym `<input>`, który może wyświetlać bieżącą wartość lub obsługuje edycję, aby osoby korzystające z technologii wspomagających mogły dokładnie nawigować i wchodzić w interakcje z treścią.


<div id="locSnippet"></div>

### Co zrobić

* Unless the `has-popup` attribute value is `dialog`, set the initial DOM focus only on the combobox’s `<input>` element.

* Jeśli wartością atrybutu `has-popup` nie jest `dialog`, ustaw początkowy fokus DOM tylko na elemencie `<input>` elementu combobox.

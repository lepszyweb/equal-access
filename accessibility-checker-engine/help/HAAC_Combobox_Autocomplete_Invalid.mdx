### The `aria-autocomplete` attribute value of `"inline"` is not valid for a combobox


### Wartość `"inline"` atrybutu `aria-autocomplete` dla pola combobox  nie jest prawidłowa

<div id="locLevel"></div>

The `aria-autocomplete` attribute value for a combobox text input element must not be `"inline"`

Wartością atrybutu `aria-autocomplete` dla elementu kombi nie może być `"inline"`

</Column>
</Row>
<Row>

### Dlaczego to jest ważne

Technologie wspomagające mogą przedstawiać użytkownikom różne typy combobox zgodnie z ich zamierzonym zachowaniem. Wartość `"inline"` atrybutu `aria-autocomplete` nie jest prawidłowa dla tekstu w polu kombi, więc technologie wspomagające mogą nie być w stanie go obsługiwać. Kombinacja, która obsługuje wbudowane formularze autouzupełniania, zrobiłaby to w połączeniu z elementem wyskakującym i powinna używać `aria-autocomplete="both"`.

<div id="locSnippet"></div>

### Co zrobić

* Jeśli kontrolka combo ma funkcję autouzupełniania, która obejmuje zarówno tekst w wierszu pojawiający się za kursorem użytkownika, jak i element wyskakujący, użyj `aria-autocomplete="both"`;
* LUB, jeśli element combobox ma zachowanie autouzupełniania zapewniane tylko przez element popup, użyj `aria-autocomplete="list"`;
* LUB, jeśli  element combobox nie ma funkcji autouzupełniania, usuń atrybut `aria-autocomplete` (wartość domyślna to `"none"`).



Na przykład:

<CodeSnippet type="multi" light={true}> &lt;label for="tag_combo"&gt;Znacznik&lt;/label&gt;
  &lt;input type="text" id="tag_combo"
      role="combobox" aria-autocomplete="list"
      aria-haspopup="listbox" aria-expanded="true"
      aria-controls="popup_listbox" aria-activedescendant="selected_option"&gt;
&lt;ul role="listbox" id="popup_listbox"&gt;
   &lt;li role="option"&gt;Zebra&lt;/li&gt;
   &lt;li role="option" id="selected_option"&gt;Powiększenie&lt;/li&gt;
&lt;/ul&gt;</CodeSnippet>
